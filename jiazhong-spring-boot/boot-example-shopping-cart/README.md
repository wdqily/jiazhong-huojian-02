## 项目简介

本项目是一个基于 Spring Boot 3 构建的购物车后端系统。项目主要实现了用户模块、商品模块以及购物车模块，并在数据持久化层面引入了Redis
缓存，通过定时任务机制实现 Redis 数据到 MySQL 的异步同步。

## 核心技术栈

**后端架构：**

Spring Boot 3+MyBatis-Plus+Druid+MySQL+Redis+Fastjson

**前端技术：**

HTML + Vue.js + Axios

## 核心及特色功能

1. **基础业务模块**
    * **用户管理 (Users)**：支持用户登录，采用jwt登录方式，密码采用 MD5 加密处理以保证基础安全性,使用`localStorage`
      保存登录用户信息。
    * **商品管理 (Goods)**：提供基础的商品数据读取和展示接口。
    * **购物车管理 (Carts)**：实现商品加入购物车、删除购物车、清空购物车、修改商品数量、购物车列表查询等功能。
2. **Redis 与 MySQL 数据同步**
    * 项目配置了定时任务 (`RedisCopyMySQLScheduled`)，用于定期将 Redis 中的购物车数据备份 MySQL 数据库中。
    * 添加购物车商品时，写入 Redis list.
    * 查询购物车时，优先查 Redis。
    * Redis 没有购物车数据时，再从 MySQL 查询。
3. **数据逻辑删除**
    * 通过 `MyBatis-Plus` 实现了全局的逻辑删除控制（对 `state`字段 ：`1` 代表未删除，`0` 代表已删除）。
4. **统一返回结果**
   *service实现类统一返回类型为`JsonResult`

## 项目问题及解决方案

1. **高并发的添加商品到购物车可能会导致雪崩**
    * 对此引入了`redis`作为缓冲层
2. **用户的密码可能暴露在前端页面**
    * 使用了MD5登录方式以保护用户隐私
3. **前端返回结果为空或报错**
    * 后端接口发生变化，前端页面接收参数也要变
4. **不能进行清空操作**
    * `controller`层传参时没加注解
